# Backlog do produto - ERP SaaS para prestadores de servicos

## 1. Objetivo

Este backlog organiza o produto em epicos, features e historias de usuario para orientar priorizacao, discovery e planejamento de entregas.

## 2. Estrutura de priorizacao

### 2.1 Legenda de prioridade

- P0: essencial para o MVP
- P1: importante para validacao comercial
- P2: importante para escala e monetizacao
- P3: expansao e sofisticacao

### 2.2 Legenda de fase

- F0: fundacao
- F1: MVP operacional
- F2: validacao comercial
- F3: financeiro e monetizacao
- F4: escala e especializacao

## 3. Epic A - Plataforma SaaS

### Feature A1 - Tenant e onboarding

Prioridade: P0  
Fase: F0

Historias:

- Como dono da plataforma, quero criar tenants para separar os dados de cada cliente.
- Como novo tenant, quero configurar dados basicos da empresa para iniciar o uso rapidamente.
- Como tenant, quero definir informacoes basicas do negocio para personalizar o sistema.

### Feature A2 - Usuarios e perfis

Prioridade: P0  
Fase: F1

Historias:

- Como administrador, quero convidar usuarios para operar o sistema em equipe.
- Como administrador, quero definir perfis de acesso para limitar a visualizacao e edicao.
- Como usuario, quero acessar apenas os modulos permitidos para meu perfil.

### Feature A3 - Auditoria basica

Prioridade: P1  
Fase: F2

Historias:

- Como gestor, quero rastrear alteracoes criticas para ter mais controle.
- Como administrador, quero saber quem mudou um status, aprovou algo ou registrou pagamento.

## 4. Epic B - Clientes e relacionamento

### Feature B1 - Cadastro de clientes

Prioridade: P0  
Fase: F1

Historias:

- Como atendente, quero cadastrar clientes rapidamente para iniciar atendimento.
- Como atendente, quero localizar clientes por nome, telefone ou documento.
- Como gestor, quero manter historico mesmo quando o cliente ficar inativo.

### Feature B2 - Contatos e enderecos

Prioridade: P0  
Fase: F1

Historias:

- Como usuario, quero registrar mais de um contato por cliente.
- Como usuario, quero salvar mais de um endereco para atendimentos futuros.
- Como usuario, quero identificar observacoes importantes do cliente.

### Feature B3 - Historico do cliente

Prioridade: P1  
Fase: F2

Historias:

- Como atendente, quero ver historico de orcamentos, servicos e cobrancas do cliente.
- Como gestor, quero entender recorrencia e comportamento do cliente.

## 5. Epic C - Catalogo de servicos

### Feature C1 - Cadastro de servicos

Prioridade: P0  
Fase: F1

Historias:

- Como administrador, quero cadastrar servicos padrao para acelerar orcamentos.
- Como usuario, quero definir valor base e duracao estimada para cada servico.
- Como usuario, quero desativar servicos sem perder historico.

### Feature C2 - Templates e padroes

Prioridade: P1  
Fase: F2

Historias:

- Como administrador, quero criar templates de servico para padronizar atendimento.
- Como usuario, quero sugerir checklist e observacoes automaticamente ao selecionar um servico.

## 6. Epic D - Orcamentos e propostas

### Feature D1 - Criacao de orcamentos

Prioridade: P0  
Fase: F1

Historias:

- Como atendente, quero criar orcamentos rapidamente para responder clientes mais cedo.
- Como usuario, quero incluir itens de servico, descontos e observacoes.
- Como usuario, quero duplicar um orcamento existente para ganhar tempo.

### Feature D2 - Envio e aceite

Prioridade: P0  
Fase: F2

Historias:

- Como usuario, quero enviar orcamento por link para facilitar aprovacao.
- Como cliente, quero aprovar ou recusar um orcamento de forma simples.
- Como gestor, quero acompanhar quais orcamentos ainda estao pendentes.

### Feature D3 - Conversao em OS

Prioridade: P0  
Fase: F1

Historias:

- Como usuario, quero transformar um orcamento aprovado em OS sem retrabalho.
- Como gestor, quero manter rastreabilidade entre venda e execucao.

Prioridade: P0  
Fase: F1
