## 3. Telas principais do sistema

### 3.1 Dashboard

Objetivo:

- mostrar visao rapida do negocio

Blocos sugeridos:

- servicos de hoje
- orcamentos pendentes
- OS em andamento
- recebimentos pendentes
- agenda do dia
- alertas importantes

### 3.2 Lista de clientes

Objetivo:

- localizar e gerenciar clientes rapidamente

Itens:

- busca
- filtros
- criar cliente
- ver historico
- acessar OS/orcamentos

### 3.3 Ficha do cliente

Objetivo:

- centralizar todas as informacoes do cliente

Abas:

- dados gerais
- contatos
- enderecos
- historico
- orcamentos
- OS
- financeiro
- anexos

### 3.4 Catalogo de servicos

Objetivo:

- padronizar operacao comercial e de execucao

Campos:

- nome
- categoria
- valor base
- duracao estimada
- checklist padrao
- observacoes internas

### 3.5 Orcamento

Objetivo:

- montar proposta rapidamente

Itens:

- cliente
- servicos
- descontos
- observacoes
- validade
- aprovar/reprovar
- converter em OS

### 3.6 Lista de OS

Objetivo:

- acompanhar operacao

Filtros:

- status
- profissional
- periodo
- cliente
- prioridade

### 3.7 Detalhe da OS

Objetivo:

- concentrar toda a execucao do servico

Blocos:

- cabecalho da OS
- timeline
- agenda vinculada
- checklist
- anexos
- observacoes
- comprovante/conclusao
- cobranca

### 3.8 Agenda

Objetivo:

- organizar compromissos e execucao

Visoes:

- agenda diaria
- agenda semanal
- agenda por profissional

Acao:

- arrastar/agendar
- reagendar
- confirmar
- marcar como concluido

### 3.9 Financeiro

Objetivo:

- controlar recebimentos e caixa basico

Secoes:

- contas a receber
- recebidos hoje
- vencidos
- fluxo previsto
- link de pagamento

### 3.10 Portal do cliente

Objetivo:

- dar autonomia e transparencia

Funcionalidades:

- ver orcamentos
- aprovar
- acompanhar status
- pagar
- baixar comprovantes
- consultar historico

## 4. Fluxos principais

### 4.1 Fluxo comercial-operacional

8. Cobranca

### 4.2 Fluxo operacional direto

1. Cliente
2. OS
3. Agenda
4. Execucao
5. Cobranca

### 4.3 Fluxo financeiro

1. Gerar recebivel
2. Enviar cobranca
3. Registrar pagamento
4. Atualizar status
5. Exibir no dashboard

## 5. Regras de UX do produto

- reduzir quantidade de campos obrigatorios
- permitir uso rapido em celular
- sempre mostrar proximo passo recomendado
- manter linguagem simples e direta
- evitar tela poluida para usuario autonomo
